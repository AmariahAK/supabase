Supabase provides everything you need to connect an AI coding agent to your project: a live connection to your database and platform (MCP), portable instructions your agent can reuse (Agent Skills), a one-step bundle of both (Plugin), and copy-paste prompts for tools that don't support any of the above.

## Get started

The fastest way to get going is the Supabase Plugin for AI Coding Agents. It bundles the MCP server and Agent Skills into a single install:

```bash
npx plugins add supabase-community/supabase-plugin
```

<ContentListings id="ai-tools-supported-agents" />

Agent Skills work with any agent that can read Markdown-based instructions — see [Agent Skills](/docs/guides/ai-tools/ai-skills) for supported formats.

### Want more control?

- Want your agent to read your schema, run queries, or apply migrations directly? Install the [MCP Server](/docs/guides/ai-tools/mcp).
- Want consistent Supabase and Postgres guidance your agent can reuse across sessions? Install [Agent Skills](/docs/guides/ai-tools/ai-skills).
- Want both MCP and Agent Skills in a single install? Use the [Agent Plugin](/docs/guides/ai-tools/plugins).
- Your agent doesn't support MCP, plugins, or skills? Copy in a ready-made [prompt](/docs/guides/ai-tools/ai-prompts) instead.

## Key concepts

- **MCP (Model Context Protocol)**: a live connection between your agent and your actual Supabase project. Once connected, your agent can call tools to query data, run migrations, deploy Edge Functions, and more.
- **Agent Skills**: portable, on-demand instructions your agent loads when it needs Supabase- or Postgres-specific procedural knowledge. Skills don't require a live connection, and work across different agents.
- **Plugin**: a single install that bundles the MCP server and Agent Skills together for a specific agent.
- **Prompts**: static prompt files you copy into your project for agents that don't support MCP, plugins, or skills natively.

## Building AI into your app?

The tools above are for your development workflow. If you're building AI capabilities into your own product:

- [Deploy MCP servers](/docs/guides/ai-tools/byo-mcp) - Host your own MCP server on Supabase Edge Functions so your users can connect their AI agents to your product
- [Vectors / Embeddings](/docs/guides/ai) - Build semantic search, RAG pipelines, and other AI-powered features using pgvector
